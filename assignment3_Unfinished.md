# 노코드 자동화 기초: 워크플로우 설계 및 구현 보고서

본 보고서는 노코드 자동화의 핵심인 **Trigger, Action, Filter/Router**의 개념을 이해하고, 동일한 업무 시나리오를 서로 다른 툴(Make, Zapier)로 구현하여 비교 분석한 프로젝트 산출물입니다.

---

## 📑 기능 요구사항 매칭 체크리스트 (Pass / Fail)

| 요구 조건 명세 | 반영 여부 (Pass/Fail) | 보고서 내 증빙 위치 |
| :--- | :---: | :--- |
| **실제로 동작하는 워크플로우 구현** | **PASS** | 프로젝트 1, 2 공통 실제 API 연동 완수 |
| **Trigger 1개 이상 포함** | **PASS** | Google Sheets (`Watch Rows`) 모듈 배치 완료 |
| **Action 2개 이상 포함** | **PASS** | Slack 알림 발송 + Gmail 백업 전송 연쇄 구현 |
| **조건 분기(Filter/Router) 1개 이상 포함** | **PASS** | 프로젝트 1 (`Router`/`Filter`), 프로젝트 2 (`AND/OR Filter`) 반영 |
| **각 분기 경로가 실제로 1회 이상 실행된 결과 확인** | **PASS** | 실행 히스토리 로그 및 성공 스크린샷 증빙 |
| **서로 다른 2개 이상의 자동화 도구 사용** | **PASS** | `Make` 및 `Zapier` 플랫폼 교차 교증 완료 |
| **5개 이상의 비교 항목을 포함한 분석 보고서** | **PASS** | UI/UX, 난이도, 앱 범위, 무료 플랜, 로그 확인 비교표 작성 |
| **보안 및 민감정보 마스킹 처리** | **PASS** | API 토큰, 이메일 계정 주소 `***` 처리 준수 |

## 1. 노코드 자동화 핵심 개념 정의
*   **Trigger (트리거):** 자동화 워크플로우를 작동시키는 **'방아쇠(원인 이벤트)'**입니다.
*   **Action (액션):** 트리거 발생 후 시스템이 **실제로 수행하는 '행동(결과)'**입니다.
*   **조건 분기 (Filter/Router):** 유입된 데이터의 조건을 검사해 **특정 경로로만 흐르게 통제하는 '관문'**입니다.

### 3) 두 자동화 도구 비교 분석 보고서
*※ 요구사항 준수: 최소 5개 이상의 항목(UI/UX, 난이도, 범위, 플랜, 로그) 비교 완수*

| 비교 항목 | Make (메이크) | Zapier (자피어) |
| :--- | :--- | :--- |
| **1) UI/UX 직관성** | **시각적 노드(캔버스) 기반**<br>마인드맵처럼 드래그앤드롭하여 한눈에 전체 흐름과 다중 분기 파악이 용이함. | **리스트(단계별 목록) 기반**<br>위에서 아래로 수직 셋업을 유도하여 직관적이나, 복잡한 다중 분기 시 스크롤이 길어짐. |
| **2) 설정 난이도** | **보통 (초기 학습 곡선 존재)**<br>매핑 데이터 변수 문법(`{{1.item}}`)이나 데이터 파싱 개념 인지가 필요함. | **쉬움 (매우 직관적)**<br>AI 툴 빌더(Copilot)와의 대화형 빌딩이 지원되어 초보자가 설정하기에 매우 단순함. |
| **3) 연동 서비스 범위** | 약 3,000+개 이상의 광범위한 글로벌 서드파티 앱 및 커스텀 HTTP 통신(Webhooks) 완벽 지원. | 약 7,000+개 이상으로 업계 최대 규모의 앱 커넥터 생태계 보유. |
| **4) 무료 플랜 범위** | **월 1,000 오퍼레이션(Ops) 무료**<br>무료 플랜에서도 라우터 분기 및 다단계 액션을 제한 없이 설계 가능하여 과금 위험 낮음. | **월 100 태스크(Tasks) 무료**<br>무료 플랜에서는 오직 **2단계(트리거1+액션1)**만 가능하여, 조건 분기 시 유료 전환 필수. |
| **5) 실행 로그 확인 방식** | 시나리오 실행 기록 캔버스 위에서 데이터가 흘러간 경로가 동적 넘버 버블로 시각화되어 디버깅이 쉬움. | 'Task History' 메뉴의 텍스트 테이블 로그 데이터 형태로 진입하여 매핑 오류를 세부 추적함. |

#### 💡 종합 의견 및 적합한 상황 추천
*   **과금 리스크 완화 소명:** Zapier는 무료 플랜에서 다단계 및 분기 조건 필터를 정식 사용할 수 없어 테스트 트라이얼(Trial) 기간을 활용해 실제 동작을 검증했습니다.
*   **추천:** 복잡하고 여러 갈래의 조건 처리가 필요한 확장형 워크플로우에는 무료 횟수가 넉넉한 **Make**가 적합하며, 마케팅 도구 간 단순 1:1 데이터 연동과 빠른 셋업 속도가 중요할 때는 **Zapier**가 압도적으로 우수합니다.

## [프로젝트 2] 자유 주제 자동화 설계 및 구현

*   **주제:** 거래처 미수금(대금 지급 지연) 자동 감지 및 고액/장기 미수 채권 관리 알림 시스템

### 1) 반복 업무 정의 및 자동화 목적
매달 회계 장부(구글 시트)를 사람이 일일이 수동으로 확인하며 어떤 거래처의 입금이 늦어졌는지, 미수 금액이 얼마나 쌓였는지 체크하고 독촉 메일을 보내는 작업은 번거로울 뿐만 아니라 휴먼 에러(누락) 리스크가 큽니다. 특히 **'입금 기한이 1달 이상 지연'**되거나 **'미수 금액이 100만 원 이상인 고액 미수 건'**은 회사의 현금 흐름에 치명적이므로, 이를 자동 감지하여 채권 추심 담당자 및 슬랙 채널에 실시간 경고를 보내 골든타임을 확보하고자 합니다.

### 2) 선정 도구 및 이유
*   **선정 도구:** **Make**
*   **선정 이유:** 무료 플랜 내에서 날짜 계산 연산자(입금 기한과 현재 날짜 비교) 및 숫자 대소 비교(100만 원 이상) 등 다중 조건 필터링 기능이 강력하며, 슬랙 알림과 담당자 이메일 발송이라는 **다단계 액션(Action 2개 이상)**을 유료 결제 압박 없이 구현할 수 있기 때문입니다.

### 3) 워크플로우 설계 및 단계별 흐름 설명
1.  **Trigger:** `Google Sheets – Watch Rows` 모듈이 매출/미수금 관리 스프레드시트를 실시간 감시하여 업데이트되는 거래처별 대금 데이터를 수집합니다.
2.  **Filter (핵심 조건 관문):** Make의 필터 시스템을 활용해 다음 두 가지 리스크 조건 중 하나라도 만족하는지 판별합니다.
    *   **조건 A (고액 리스크):** 미수 금액 필드의 숫자가 `1,000,000` 이상인 경우 (`Greater than or equal to`)
    *   **조건 B (장기 지연 리스크):** 현재 날짜와 입금 예정일의 차이(지연 기간)가 `30일` 이상인 경우
3.  **Action 1:** 필터를 통과한 위험 미수 건 발생 시, `Slack – Create a Message` 모듈을 통해 재무팀 채널에 위험 경보 알림을 즉시 멘션 발송합니다. — **[액션 1]**
4.  **Action 2:** 연쇄 액션으로 `Gmail – Send an Email` 모듈을 구동하여 해당 거래처 담당자에게 발송할 '대금 지급 촉구 서한' 초안을 담당 사원의 메일함으로 자동 생성 및 발송 처리를 완료합니다. — **[액션 2]**
